 For our organisation, Patrick and Slackware has consistently been a rock solid implementation of the Linux Kernel with the GNU tool chain and application packages. What's really important for the systems we deploy is the ability to make customised hardened systems. I know that many prefer kernels with module support although the bulk of out deployed systems use monolithic kernels built specifically for the hardware which only changes when something breaks. The ability to replace the kernel from the delivered sources (or new versions) is very handy. As with any server class system uptime is paramount and Slackware continually provides this in every release.

On the whole the Slackware distribution allows for maximum configuration with minimal system overhead. Whilst it may seem easier to use a package management system that includes dependency tracking it's never been an issue for us. Yes Slackware does not provide a warm hand holding experience, it does instead provide a solid base for learning and expanding the knowledge about the software used in the deployment.

Pros:
- Stability and reliability of the base system.
- Clarity of implementation and management.
- Ability to extend and enhance the basic platform
- Building application packages is actually relatively easy from source.
- A vibrant third party package community, with plenty of help available.

Cons:
- Some applications require some digging to get them to build (not a newbie task)
- System versions are sometimes a long time between releases.

 I came to Slackware via Salix which is a great distro for those who may be intimidated by the supposed complexity of installing and maintaining Slackware (eg. lack of dependency checking seemed entirely alien to me).

With 2 years under my belt running on a desktop and laptop at home I am glad to report it is far easier than I thought. No mishaps at all and I am totally converted. I never again want to do battle with a "real" package manager again.

Slackware really does take up a lot less time to maintain.
Documentation is excellent as is help when it comes from the Slackware community on different forums. My impression is that error messages are far more explicit than some other distros often allowing you to fix the issue without an internet search.

Other criticisms of Slackware I can dispell are the lack of packages and how quickly they are updated. In pure quantity of packages the official repository may be considered lacking but the point is they give you a fully functioning system to build upon.

The Slackbuilds repository really has everything I've looked for. And there are some good pre-built 3rd party package repositories easy to use with Slackpkg+. You also have the tools to adapt or write Slackbuilds scripts to build exactly the package you need - which can be very helpful when it comes to hardware drivers.

When I've tried other distros recently I often find my Slackware system is running a newer version sometimes even compared to rolling-release distros. Security updates are particularly prompt.

I cannot recommend highly enough. Just give yourselves time for the initial installation, and from then on Slackware is likely to be a massive time saver.
